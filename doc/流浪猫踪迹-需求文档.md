# 🐾 流浪猫足迹 · 小程序需求设计文档

---

## 1. 项目概述

### 1.1 项目名称
**流浪猫足迹**（Stray Cat Tracks）

### 1.2 项目目标
打造一款轻量级微信小程序，帮助用户记录公园、城中村、校园等场景中的流浪猫。用户可拍摄猫咪照片、录制声音，并为每只猫建立专属档案，档案包含完整的属性清单，便于长期追踪与关爱。

### 1.3 目标用户
- 关爱流浪猫的普通市民
- 社区流浪猫救助者
- 校园动物保护社团成员

### 1.4 核心价值
- 以“一猫一档”的形式建立可视化猫咪身份卡
- 整合照片与声音，丰富档案维度
- 操作极简，3分钟内完成一只猫的建档

## 2. 功能需求总览

### 2.1 功能模块划分
| 模块 | 功能点 | 优先级 |
|------|--------|--------|
| 猫咪档案管理 | 新建档案、编辑档案、查看档案列表、档案详情 | P0 |
| 多媒体采集 | 拍摄/上传照片、录制/上传猫咪叫声 | P0 |
| 猫咪属性体系 | 预定义属性标签选择 + 自由文本描述 | P0 |
| 位置标记 | 自动获取当前位置、手动选址 | P1 |
| 简单统计 | 个人记录的猫咪总数、最后遇见时间线 | P2 |

### 2.2 不包含的功能（V1.0）
- 社交分享、评论、点赞
- 多人协作编辑同一只猫档案
- 猫咪识别AI、品种自动鉴定
- 消息通知、领养信息发布

## 3. 页面结构与导航

### 3.1 页面清单
| 页面路径 | 页面名称 | 功能说明 |
|----------|----------|----------|
| pages/index/index | 首页·猫猫地图 | 显示用户记录过的猫咪卡片列表，按最近遇见排序 |
| pages/add/add | 添加新猫咪 | 填写属性、拍摄照片、录制声音，创建档案 |
| pages/detail/detail | 猫咪详情 | 展示完整属性、照片画廊、声音播放、编辑入口 |
| pages/edit/edit | 编辑档案 | 修改属性、增删照片、重新录音 |
| pages/record/record | 录音界面 | 专门的声音录制与试听页面（可从添加/编辑页跳转） |

### 3.2 导航方式
- 底部 TabBar（2项）：
  - **遇见**（首页，图标：猫爪）
  - **记录**（跳转添加新猫咪，图标：加号）
- 其他页面通过导航栏返回或页面内按钮跳转。

## 4. 猫咪完整属性清单

每只猫档案包含以下字段，需支持必填与选填区分。

| 属性分类 | 属性名称 | 类型 | 必填 | 说明 |
|----------|----------|------|------|------|
| 基础身份 | 猫咪昵称 | 文本(20字) | 是 | 用户为猫起的名字，如“橘子” |
|  | 遇见日期 | 日期时间 | 是 | 首次记录日期，默认当前时间 |
|  | 遇见地点 | 地理位置 | 是 | 可自动定位或手动选择，含名称与坐标 |
| 外貌特征 | 毛色 | 多选标签 | 是 | 预设：橘色、黑白、纯黑、纯白、三花、狸花、玳瑁、灰狸、其他 |
|  | 毛长 | 单选标签 | 否 | 短毛、长毛、无毛 |
|  | 体型 | 单选标签 | 否 | 瘦小、正常、偏胖、怀孕中 |
|  | 显著特征 | 文本(50字) | 否 | 如“左耳缺角”、“尾巴折断”、“戴红色项圈” |
|  | 眼部颜色 | 单选标签 | 否 | 黄色、蓝色、绿色、异瞳、其他 |
| 健康状态 | 健康状况粗略评估 | 单选标签 | 否 | 健康、轻伤、明显疾病、未绝育(有耳标则勾选已绝育) |
|  | 健康状况备注 | 文本(100字) | 否 | 自由描述伤口位置、行为异常等 |
| 性格行为 | 性格标签 | 多选标签 | 是 | 亲人、警惕、胆小、凶猛、话痨(爱叫)、安静、贪吃、独行侠 |
|  | 与其他猫关系 | 文本(100字) | 否 | 描述与其他流浪猫的互动 |
| 声音记录 | 猫咪叫声 | 音频文件 | 否 | 通过录音或上传获得，可多个 |
|  | 叫声描述 | 文本(50字) | 否 | 如“沙哑”、“洪亮”、“咕噜声” |
| 影像记录 | 照片 | 图片文件 | 是 | 至少1张，最多9张，支持现场拍摄或相册选择 |
| 其他 | 备注故事 | 文本(200字) | 否 | 关于这只猫的额外故事或投喂心得 |

## 5. 页面详细设计

### 5.1 首页·猫猫地图 (`pages/index/index`)

**布局：**
- 顶部：标题“我遇见的猫咪” + 计数“共 X 只”
- 主体：纵向卡片列表，每张卡片为一只猫的缩略信息
- 空状态：插画 + “你还没有记录任何猫咪，点击下方 + 开始吧”

**单张卡片内容：**
- 封面照片（第一张照片）
- 昵称、毛色标签、遇见地点简写
- 距离首次遇见的天数（如“3天前遇见”）
- 卡片点击 → 跳转详情页

**排序规则：**
- 默认按`遇见日期`降序，最近遇见的在前

### 5.2 添加新猫咪 (`pages/add/add`)

**结构：**
- 分步骤表单，视觉引导清晰，但可一步完成（非强分步）
- 区块划分：
  1. **照片**：九宫格上传区，大加号按钮，支持相机/相册
  2. **声音**：录音按钮，点击跳转录音页，录制完成后返回并显示音频条
  3. **属性区**：折叠面板展示属性标签，滚动选择
  4. **遇见地点**：地图选点组件，默认当前定位，可搜索地址
  5. **昵称与备注**：输入框

**提交逻辑：**
- 点击“保存档案”按钮，校验必填项（照片≥1张、昵称、毛色、性格、地点），通过后保存至本地存储与云数据库。
- 保存成功 → 跳转至该猫咪详情页，并 toast 提示“档案建立成功”。

### 5.3 猫咪详情 (`pages/detail/detail`)

**结构：**
- 顶部：照片轮播（可全屏预览）
- 属性信息卡片：分类展示所有属性，标签用彩色圆角方块
- 声音区域：播放按钮列表，点击播放音频
- 遇见位置：地图静态截图，显示地点名称，可点击打开地图导航
- 底部操作栏：编辑档案按钮、删除按钮（二次确认）

### 5.4 编辑档案 (`pages/edit/edit`)

复用添加页面的表单结构，预填已有数据。可修改所有字段，支持增删照片、重新录音。保存后更新详情页。

### 5.5 录音界面 (`pages/record/record`)

**界面：**
- 居中圆形录音按钮，长按录音，松开结束
- 录音时长实时显示
- 下方试听列表：本次录制的片段，可试听、删除、重录
- 确认按钮，返回并携带音频文件临时路径

**权限：**
- 首次使用时需授权麦克风权限

### 6.2 本地存储与云端
- **用户维度**：每个用户拥有自己的猫咪列表，云端存储通过 `_openid` 隔离

- 照片与音频文件存储于云存储，路径格式：`{openid}/cats/{catId}/photos/` 和 `sounds/`

## 7. 交互与异常状态处理

| 场景 | 处理方式 |
|------|----------|
| 网络断开 | 档案数据先存本地缓存，网络恢复后同步云端；上传文件时提示离线状态 |
| 定位失败 | 允许手动选择地图位置或输入地点名称 |
| 麦克风权限拒绝 | 录音按钮置灰，点击时弹出“请在设置中开启麦克风权限”提示 |
| 相册权限拒绝 | 引导用户进入设置开启，否则仅可拍照 |
| 照片数量超限 | 超过9张时隐藏添加按钮并提示 |
| 必填项未填 | 提交时表单校验，高亮未填项，页面滚动至第一处缺失位置 |
| 文件上传失败 | 自动重试一次，失败后标记为“待上传”，可在编辑页面重新上传 |
| 删除档案 | 弹出确认弹窗：“确定删除这只猫咪的所有记录吗？”，确认后物理删除 |

## 8. 技术实现建议（面向Claude开发）

### 8.1 推荐技术栈
- **框架**：微信小程序原生框架（或 Taro + React/Vue，视熟悉度而定）
- **UI 库**：WeUI 扩展库或 Vant Weapp，便于快速构建表单与卡片
- **后端服务**：微信云开发（云函数 + 云数据库 + 云存储），免服务器运维
- **音频录制**：`wx.getRecorderManager()` API
- **图片选择**：`wx.chooseMedia` 或 `wx.chooseImage`
- **地图组件**：`<map>` 组件及 `wx.chooseLocation`

### 8.2 Claude 开发辅助点
- 请直接让Claude生成具体的页面 `.wxml`、`.wxss`、`.js` 代码，按页面逐一提供
- 提供上述 JSON 数据结构作为接口设计参考，可让Claude生成云函数操作数据库的代码
- 属性标签的枚举值可写入一个常量文件 `constants/cat-tags.js`，便于维护

### 8.3 开发顺序建议
1. 初始化项目结构，配置云开发环境
2. 实现添加页面（不含录音）—— 完成表单与图片上传
3. 实现首页列表与详情页
4. 加入录音功能与声音播放
5. 完善编辑、删除、定位
6. 异常处理与 UI 打磨

## 9. 验收标准

1. 用户能在 2 分钟内成功创建一只猫的完整档案（含照片、声音、地点、属性）
2. 所有必填属性标签均可通过选择器完成，无需手动输入标签
3. 录制的猫咪叫声可试听且播放流畅
4. 卡片列表正确按时间排序，点击可进入详情查看所有信息
5. 档案编辑后数据及时同步，删除后彻底消失
6. 在无网络环境下，核心浏览功能不受影响（已缓存数据）
